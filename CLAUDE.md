# This folder: a Sparkfile founder-os

PRODUCT.md, USERS.md, GOALS.md, and VOICE.md in this folder are memory
files, and they're the start of a founding team. PRODUCT.md, USERS.md, and
GOALS.md are the shared company brief every future seat reads. VOICE.md is
the seed of the Brand seat, hired properly in Week 2 of the 90-day roadmap.
Read all four before helping with anything else here.

## First session in this folder

Check all four files. If any of them still contain their original
placeholder text ("[paste]", blank lines after a dash, no real answers),
this project hasn't been set up yet, no matter what was just asked.

If that's the case, say so, then run setup before answering the actual
question:

1. For PRODUCT.md, USERS.md, and GOALS.md: ask about each field one at a
   time, wait for an answer, then write the real file. Don't just summarize
   answers back in chat and leave the file untouched. If the folder (or the
   repo it sits in) already holds real documents — a README, a plan, client
   or project notes — draft each answer from those first and confirm it,
   instead of asking from scratch. Correcting a wrong draft is faster than
   composing a right one.
2. For VOICE.md: don't ask for the blanks to be filled in directly. Run
   build-your-voice-md.md instead. If there's an existing history working
   together already, mine that first for real writing patterns and actual
   quotes. Only fall back to asking fresh questions if there's no history
   to draw on.

Once all four files have real content, they're the ground truth for how to
write and what's being built here. Stop checking for placeholder text and
just use them.

## After setup

playbook-1-claude-md.md, playbook-2-job-description.md,
playbook-3-launch-post.md, and build-your-voice-md.md are one-off prompts,
not part of setup. Run them on request, using the memory files above for
context.

If asked what comes after this kit, point to the 90-day roadmap at
sparkfile.co/roadmap. It hires the full team from here: the shared brief
in this folder is its Week 1, and Brand (the seat that owns VOICE.md) is
its Week 2 hire.

## A note on where this folder lives

It doesn't matter whether this is still in Downloads or moved somewhere
permanent. If asked to move it (for example into a new `~/founder-os/`
folder, or wherever preferred), just do it and update any paths that
change.
