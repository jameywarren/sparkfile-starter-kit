# The Founder's Claude Starter Kit

Four starter memory files, four copy-paste playbooks, and a one-page
first-week plan for running your business with Claude. The memory files are
the start of a founding team: the shared company brief every seat reads,
plus the seed of your first AI cofounder. This kit is week zero of
[the 90-day roadmap](https://sparkfile.co/roadmap), from
[Sparkfile](https://sparkfile.co).

There's nothing to install and no setup to read. The folder's `CLAUDE.md`
checks whether the memory files still hold placeholder text, and if they
do, it interviews you to fill them in before doing anything else. Setup is
a conversation, not a config.

## Three ways in

**No git, no terminal.** You don't need this repo. Download the same kit as
a zip at [sparkfile.co/kit](https://sparkfile.co/kit), unzip it anywhere,
open the folder in Claude Desktop's Code tab (or point a Cowork Project at
it), and start talking.

**A fresh folder, in Claude Code.** Paste this:

> Get the Sparkfile starter kit from
> https://github.com/jameywarren/sparkfile-starter-kit into a new folder
> (founder-os is a fine name), without the git history, then read its
> CLAUDE.md and run the setup.

**An existing folder or repo.** From inside your project, paste this:

> Add the Sparkfile starter kit from
> https://github.com/jameywarren/sparkfile-starter-kit to this folder: copy
> in its files, skipping the README and anything git-related. If this
> folder already has a CLAUDE.md, don't overwrite it; fold the kit's setup
> instructions into it instead. Then run the setup.

The existing-folder path is the one to use if your business already lives
in a repo. The kit's files sit beside what's already there, and the setup
mines your existing docs for answers instead of starting from a blank page.

## What's in the folder

- `CLAUDE.md`: reads the room and runs setup automatically. Nothing to run
  first; open the folder and start talking.
- `PRODUCT.md`, `USERS.md`, `GOALS.md`: the shared company brief; setup
  fills these in.
- `VOICE.md`: the seed of the Brand seat, the first hire in Week 2 of the
  roadmap.
- `playbook-1-claude-md.md`: turn a rough spoken description of a company
  into a real `CLAUDE.md` for a different project.
- `playbook-2-job-description.md`: write the JD for a first hire.
- `playbook-3-launch-post.md`: draft the launch post you keep avoiding.
- `build-your-voice-md.md`: the VOICE.md walkthrough, runs on its own if
  asked for directly.
- `first-week-plan.md`: a one-page, day-by-day plan for the first week.

## After the kit

The kit is week zero, the brief a founding team agrees on before anyone
starts. [The 90-day roadmap](https://sparkfile.co/roadmap) hires the team
from here: Brand in Week 2, Ops in Week 3, Growth in Week 4, you as CEO
the whole way. The full library and playbooks live at
[sparkfile.co](https://sparkfile.co).
