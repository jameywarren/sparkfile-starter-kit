# Your first week on Claude

One page. Do these in order; each day builds on the last. By day 7 you'll
have the shared brief a founding team works from (PRODUCT, USERS, GOALS)
plus the seed of its first hire (VOICE).

**Day 1: Open this kit in Claude Desktop.**
Use the Code tab, not the terminal; nothing separate to install. Open the
folder this kit is in, wherever it currently sits, and just start talking.
The `CLAUDE.md` file in this folder notices the memory files below are
still blank and walks through setting them up before anything else.

**Day 2: PRODUCT.md and USERS.md get filled in.**
First draft, not perfect. What you're building, who it's for, what stage
you're at. Who your users actually are and what they complain about most.

**Day 3: GOALS.md gets filled in.**
Your top 3 priorities this quarter. What "on track" looks like right now.
Keep it short; this is a filter for later work, not a strategy memo.

**Day 4: VOICE.md gets filled in.**
This one runs as a real back-and-forth (via `build-your-voice-md.md`), not
a form. Real writing samples, a short avoid list, done.

**Day 5: Test it.**
Ask a question only someone who'd actually read your files could answer
well: "what's our biggest risk right now, per GOALS.md?" A generic answer
means the files are still thin. That's normal; note what's missing and
add it.

**Day 6: Run one playbook from this kit.**
Pick whichever of the three matches something real on your plate this
week. Run it once, by hand. Read the output critically against what you
actually know to be true; don't just accept a confident-sounding draft.

**Day 7: Decide what's next.**
If this went well, the 90-day roadmap at sparkfile.co/roadmap is the next
stop. Its Week 1 writes the shared brief (PRODUCT.md, USERS.md, GOALS.md)
and its Week 2 hires Brand, the seat that owns VOICE.md, so you arrive
with a head start on both. If a file still feels thin, spend ten more
minutes on it before moving on. The tuning happens in the files, not in a
longer prompt you retype every morning.
